Primitive Types

Number: Any integer or floating-point numeric value

String: A character or sequence of characters delimited by either single or double quotes

Boolean: true or false

Identifying Primitive Types

using typeof operator


typeof 100 // "number"
typeof 200.35 // "number"
typeof 'Shravan Kasagoni' // "string"
typeof false // "boolean"
typeof true // "boolean"
typeof undefined // "undefined"
typeof null // "object"

Working with null


typeof null; //the result is "object"

It's an error in JavaScript language itself, acknowledged by TC39 committee.

Reference Types

Reference types represent objects in JavaScript

An object is an unordered collection of properties

Property consists of a name (always a string) and a value (Primitive, Reference)

If value of a property is a function, it is called a method

Built-in Objects

  • Array
  • Error
  • Function
  • Math
  • Object
  • RegExp

Creating Objects

  • Object can be created using Literal Form, new operator with a constructor

Creating Object using new operator with a constructor


var person = new Object();

JavaScript objects are dynamic - properties can be added and deleted any time.


person.name = 'Shravan';
person.age = 20;
person.printDetails = function () {
  console.log(this.name, this.age);
};

Literal Form

  • A literal is syntax that allows you to define a reference value without explicitly creating an object, using the new operator and the objectâ€™s constructor
  • Several built-in reference types have literal forms

Creating Object using literal form


var empty = {};

  firstName: 'Shravan Kumar',
  lastName: 'Kasagoni'

var person = {
  firstName: 'Shravan Kumar',
  lastName: 'Kasagoni',

    console.log(this.firstName, this.lastName);

Creating Arrays using new operator with a constructor


var primaryColors = new Array("red", "green", "blue");

console.log(primaryColors[1]); // "green"

Creating Arrays using literal form


var primaryColors = ["red", "green", "blue"];

console.log(primaryColors[0]); // "red"
